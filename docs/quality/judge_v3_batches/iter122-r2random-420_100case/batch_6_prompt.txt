# CRITICAL RULES (read before scoring — these eliminate inter-judge variance)

**RULE A — score the response, not the tool choice.**
The agent decided the tool + args; the response is what the server returned. If the wrong tool was called (e.g. get_japan_heritage for a tea-ceremony query), you may note it in `improvement_hint`, but the dim scores must reflect the actual data returned, not what would have come from a different tool. A wrong-tool response with authoritative data is still partly useful — score 3, not 2.

**RULE B — single root cause, single dim hit.**
If one shortfall affects multiple dims (e.g. "wrong tool" → could affect intent / specificity / constraint), pick the ONE worst-affected dim and score that strictly; the other affected dims drop by AT MOST 1 from their otherwise-deserved score. Do NOT compound the same defect across 3-4 dims — that produces unstable inter-judge medians.

**RULE C — authoritative-source-but-incomplete = 3, not 2.**
If the response returns real, authoritative data (R-3 designations, Wikidata heritage entities, MAFF GI records, official scrapes) but the scope or filtering is incomplete (e.g. only Akita instead of all-Tohoku, or only food when craft was also requested), the affected dims are **3 (acceptable but weak)**, not 2. Score 2 only when the data is actively misleading or the relevant entity class is entirely absent.

**RULE D — language mismatch is constraint failure, not hallucination.**
If the user query is in fr / de / ko / vi / etc but the response is in en / ja, score `constraint_handling` -1 from its otherwise-deserved value. Do NOT also drop intent / specificity / expression on the same grounds — that is double-counting per Rule B.

**RULE E — featured_cluster present and relevant = boost intent / specificity by 1.**
If the response includes a `featured_cluster` block whose entries match the query toponym, the cluster delivers canonical sub-spots regardless of ranking. Add +1 to intent and specificity (capped at 5), unless the cluster items are wrong/stale.

  **5** = exact match for the dim's question, no caveats needed
  **4** = match with ONE notable caveat (mention it in reasoning)
  **3** = partial match — authoritative data with scope/filter gap (RULE C)
  **2** = wrong scope OR major gap OR misleading
  **1** = wrong tool, no useful data, or off-topic
  **0** = empty / error response
